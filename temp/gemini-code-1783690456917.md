# Loot Tracker — RPG Data Aggregation & Analytics Dashboard

A data-driven full-stack web application designed for tracking, aggregating, and analyzing loot drops, character progression, and boss encounter metrics. Developed as part of a Bachelor of Engineering (B.Eng.) thesis to demonstrate efficient relational database design and reactive state management.

## 🚀 Core Features

* **User Authentication:** Secure user registration and session management powered by JSON Web Tokens (JWT).
* **Dynamic Character & Boss Management:** Relational architecture allowing users to manage multiple characters and map them to specific bosses and difficulty scaling.
* **Granular Loot Tracking:** Structured recording of various drop categories (Items, Gold, Rars, Synergetics, Drifts).
* **Data-Driven Analytics:** Real-time calculation of drop statistics, distribution curves, and progression summaries per character/boss encounter.
* **Responsive UI:** Modern, user-centric interface focused on minimizing friction during high-frequency data logging.

## 🛠️ Engineered Tech Stack

* **Frontend:** `Vue.js` — Core reactive framework utilized for modular component architecture and efficient client-side state management.
* **Backend:** `Node.js` (`Express.js`) — RESTful API architecture ensuring clean separation of concerns, optimized routing, and request validation handling.
* **Database:** `PostgreSQL` — Relational database engine chosen for strong data integrity, acid compliance, and optimized relational schemas connecting users, characters, and drop entities.
* **Authentication:** `JWT (JSON Web Tokens)` — Stateless authentication mechanism for secure and lightweight endpoint protection.
* **DevOps & Containerization:** `Docker` & `Docker Compose` — Used to containerize the application environment, guaranteeing cross-platform consistency and reliable multi-container orchestration.
* **Version Control:** `Git` — Utilized for feature-branch workflow and tracking development milestones.
